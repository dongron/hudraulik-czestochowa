# Implementation Plan: Plumber Landing Page

**Branch**: `001-plumber-landing-page` | **Date**: 2026-05-01 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `specs/001-plumber-landing-page/spec.md`

## Summary

Build a single-page Polish-language landing site for "Usługi Hydrauliczne" — a plumbing company in Częstochowa. The page is content-managed via Sanity (5 new page-builder block types + settings extension), rendered as a static Next.js App Router page, and optimised for mobile-first SEO and emergency lead conversion. Phone call is the primary conversion action; a mocked contact form handles price-estimate requests.

## Technical Context

**Language/Version**: TypeScript 5, Next.js 16 App Router, React 19  
**Primary Dependencies**: Sanity v5, next-sanity, Tailwind CSS v4 (PostCSS mode), @portabletext/react  
**Storage**: Sanity Content Lake (cloud)  
**Testing**: None configured (deferred; not in spec scope)  
**Target Platform**: Web — Vercel (frontend) + Sanity-hosted Studio  
**Project Type**: Web application (Next.js + Sanity CMS, monorepo)  
**Performance Goals**: Google PageSpeed Insights ≥ 85 on mobile (SC-004)  
**Constraints**: No `any` types; generated `sanity.types.ts` as source of truth; `'use client'` only for interactive form; all content in Sanity (no hardcoded copy in components)  
**Scale/Scope**: Single landing page (~7 sections), 1 Sanity page document, 1 settings singleton
